# SPV 구조 + 자금 흐름 (전체 그림)

이제 PF 구조의 핵심인
👉 SPV 구조 + 전체 자금 흐름을 “한 번에 이해되는 그림 + 설명”으로 정리해드릴게요.

📌 1️⃣ SPV란?

👉 SPV (Special Purpose Vehicle)
= 특정 목적(유동화/투자)을 위해 만든 별도 법인

✔ 왜 필요할까?

👉 핵심 목적 3가지

리스크 분리 (시행사와 분리)
자금 조달 (증권 발행)
현금흐름 통제 (Waterfall)

📊 2️⃣ 전체 구조 (한눈에 그림)

ABCP / ABS 매입
   ↓

     SPV

   ↓

   ↓
──────────────
     SPV
──────────────
   ↓ (Waterfall)
투자자 상환
📌 3️⃣ 참여자별 역할
✔ ① 투자자
ABCP / ABS 매입
자금 제공
✔ ② SPV (핵심🔥)

👉 돈의 허브

자금 수취
대출 실행
현금흐름 관리
상환 집행

✔ ③ 시행사 (Developer)

사업 진행
분양 수행

✔ ④ 시공사

공사 수행

✔ ⑤ 금융기관

유동성 지원
신용보강

📌 4️⃣ 자금 흐름 (단계별)

① 자금 조달 단계

투자자 → SPV → PF 대출 실행
ABCP / ABS 발행
자금 확보

② 사업 진행 단계

SPV → 시행사 (공사비 지급)
공사 진행
비용 집행

③ 수익 발생 단계

분양/임대 → SPV 계좌 유입
가장 중요한 단계🔥

④ 상환 단계 (Waterfall)

SPV → 투자자 상환

순서:

비용 → 이자 → 원금 → 후순위 → 시행사

📊 5️⃣ 핵심 구조 특징

✔ 1. “돈은 SPV로만 흐른다”

👉 시행사가 직접 돈 못 만짐

✔ 2. “계좌 통제”

신탁계좌 / 관리계좌 사용

✔ 3. “자동 분배”

👉 Waterfall 룰로 자동 지급

📌 7️⃣ 리스크 포인트 🔥

✔ ① 분양 실패

현금 유입 ↓ → 상환 불가

✔ ② 롤오버 실패

ABCP 차환 실패 → 즉시 디폴트

✔ ③ 비용 증가

공사비 ↑ → 수익성 ↓

📊 8️⃣ 구조의 본질

👉 SPV 구조는 결국:

돈 들어오는 통로 하나로 모아서
→ 정해진 순서로 나눠주는 시스템

📌 🔟 한 줄 정리

👉
SPV = “돈을 모으고 → 통제하고 → 순서대로 나눠주는 중앙 컨트롤 타워”

🚀 핵심 한마디

👉
PF 구조의 본질은 사업이 아니라
“현금흐름을 통제하는 구조”다

원하면 다음 단계도 이어서 가능합니다:

👉 실제 PF 딜 구조 사례 (숫자로 풀어보기)
👉 Waterfall SQL 구현
👉 리스크 대시보드 설계 (실무 화면)
