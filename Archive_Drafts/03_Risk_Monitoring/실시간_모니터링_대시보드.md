# 실시간 PF 모니터링용 Cash Flow + 리스크 대시보드 설계

---

## 1. 목적

- PF 프로젝트의 **현금흐름과 리스크를 실시간 감시**
- 조기 경보(Alert) 발동
- 투자자/은행/증권사 의사결정 지원

---

## 2. 전체 구조 (실시간 흐름)

```text id="pf-dashboard-flow"
[데이터 소스]
   │
   │ Kafka / Streaming
   ▼
[Feature 계산 엔진]
   │ DSCR, LTV, 분양률, 금리 민감도
   ▼
[ML 부도 예측 (LightGBM)]
   │ Default Probability 계산
   ▼
[Risk Scoring & Alert 판단]
   │ Risk Grade 산출
   ▼
[DB 저장 / 캐싱 (Redis)]
   ▼
[대시보드 시각화 / 알림]

4. 대시보드 KPI 구성
구분	항목	설명
프로젝트 상태	DSCR / LTV	실시간 안정성 지표
부도 예측	Default Probability	LightGBM 모델 산출
위험 등급	Risk Grade	AAA~High Risk
알림	Alert Trigger	DSCR<1, LTV>80%, Default Prob>30%
Cash Flow	월별/주별	현금 흐름 시계열
분양/진행	Sales Rate, Progress Rate	실시간 상태 추적

7. 대시보드 레이아웃 (예시)
┌───────────────────────────────┐
│ 프로젝트 선택                │
├─────────────┬─────────────────┤
│ Cash Flow   │ DSCR / LTV       │
│ 차트        │ 선 그래프       │
├─────────────┴─────────────────┤
│ Default Probability (ML)      │
│ Risk Grade 시각화             │

└───────────────────────────────┘
8. 실시간 처리 로직
Kafka / Streaming으로 프로젝트 데이터 수신
Feature 계산 (DSCR, LTV, Cash Flow)
ML 모델로 Default Probability 예측
Risk Grade 산출
Alert 조건 판단 → 알림 발송
모든 데이터 DB/Redis 저장 → 대시보드 갱신
9. 운영 고려사항
Latency: <1초 실시간 처리
Scalability: 프로젝트 수 증가 대응
Failover: Kafka / API 서버 장애 대비
Security: 금융 데이터 보호, 인증/권한 관리
10. 한눈에 요약
데이터 → Feature 계산 → ML 예측 → Risk Grade → Alert → 대시보드 시각화
             ↑
             └ 리스크 포인트: DSCR/LTV 저하, 분양률 저조, Cash Flow 부족
