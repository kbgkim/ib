# LightGBM + SHAP 상세 개념 정리 (그림/예시 포함)

------------------------------------------------------------------------

# 1. 전체 개념 한눈에 보기

    [Feature 입력]
       ↓
    LightGBM (예측)
       ↓
    성공 확률 (0~1)
       ↓
    SHAP (설명)
       ↓
    각 Feature 영향도

------------------------------------------------------------------------

# 2. Feature란 무엇인가

Feature = 모델이 판단하는 근거

예시 데이터:

  Feature         값
  --------------- -----
  discount_rate   15%
  debt_ratio      80%
  size_ratio      0.3
  market_return   +5%

👉 사람 판단: - 할인율 높음 → 긍정 - 부채 높음 → 부정

👉 모델도 동일하게 학습

------------------------------------------------------------------------

# 3. LightGBM 동작 원리 (그림)

## 3-1. 단일 트리

              discount_rate > 10?
                /        \
             YES         NO
            /              \
       성공확률↑        실패확률↑

------------------------------------------------------------------------

## 3-2. 여러 트리 결합

    Tree1 → 0.6
    Tree2 → 0.8
    Tree3 → 0.7

    최종 평균 → 0.7

👉 수백 개 트리 → 더 정확

------------------------------------------------------------------------

# 4. 왜 LightGBM이 강력한가

-   비선형 관계 처리
-   Feature 간 상호작용 반영
-   빠른 속도

------------------------------------------------------------------------

# 5. 문제점 (블랙박스)

    입력 → 모델 → 결과 (0.82)

👉 왜 0.82인지 모름

------------------------------------------------------------------------

# 6. SHAP 개념

SHAP = 결과를 Feature별로 분해

------------------------------------------------------------------------

# 7. SHAP 직관적 그림

    기본값: 0.50

    discount_rate   +0.20
    debt_ratio      -0.10
    market_return   +0.12

    -----------------------
    최종: 0.72

------------------------------------------------------------------------

# 8. 실제 예시

입력:

    discount_rate = 15%
    debt_ratio = 80%
    size_ratio = 0.3

결과:

    성공 확률: 0.78

SHAP 설명:

  Feature         영향
  --------------- -------
  discount_rate   +0.25
  debt_ratio      -0.18
  size_ratio      -0.05

------------------------------------------------------------------------

# 9. 해석 방법

-   -   값 → 성공 확률 증가
-   -   값 → 실패 방향

👉 영향 절대값이 클수록 중요

------------------------------------------------------------------------

# 10. 전체 흐름 (실무)

    1. 공시 수집
    2. Feature 생성
    3. DB 저장
    4. LightGBM 예측
    5. SHAP 설명 생성
    6. 결과 제공

------------------------------------------------------------------------

# 11. 실무에서 중요한 포인트

## 11-1. Feature 품질

모델보다 중요

## 11-2. 데이터 누수 금지

딜 이후 데이터 사용 금지

## 11-3. 설명 가능성

금융에서는 필수

------------------------------------------------------------------------

# 12. 잘못된 이해

❌ Feature 많으면 좋다\
→ 의미 있는 Feature가 중요

❌ 모델이 답을 안다\
→ 패턴만 학습
