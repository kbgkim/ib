PF(Project Financing)에서 LTV/DSCR보다 더 핵심적으로 보는 지표가 바로
👉 분양률 / IRR / NPV입니다.

실무 기준으로 개념 → 계산 → 해석 → 위험 신호까지 정리해드릴게요.

📌 1️⃣ 분양률 (Sales Rate)
✔ 개념

👉 전체 공급 대비 얼마나 분양됐는지

Sales Rate = 𝑆𝑜𝑙𝑑 𝑈𝑛𝑖𝑡𝑠 / 𝑇𝑜𝑡𝑎𝑙 𝑈𝑛𝑖𝑡𝑠

✔ 예시
총 1,000세대
분양 700세대

👉 분양률 = 70%

✔ 실무 기준

분양률	의미
80%↑	안정
60~80%	관리 필요
60%↓	위험

✔ 왜 중요한가?

PF는 분양대금으로 대출 상환
즉, 분양률 = 현금 유입 지표

✔ 위험 신호 🚨

초기 분양 저조 → 전체 프로젝트 실패 가능
특정 구간 정체 (미분양 증가)

📌 2️⃣ IRR (Internal Rate of Return)

✔ 개념

👉 투자 수익률 (연 복리 기준)

𝑁𝑃𝑉 = 0 ⇒ 𝐼𝑅𝑅

✔ 의미

“이 프로젝트가 몇 % 수익을 내는가”
투자 판단의 핵심

✔ 예시

투자: -100억
회수: 130억

👉 IRR ≈ 8~12% 수준 (기간에 따라 다름)

✔ 실무 기준

IRR	의미
15%↑	매우 우수
10~15%	양호
7~10%	보통
7%↓	투자 부적합

✔ PF에서 중요 포인트

사업기간 길수록 IRR ↓
초기 투자 구조 중요

✔ 위험 신호 🚨

IRR 하락 → 수익성 악화
비용 증가 / 분양 지연

📌 3️⃣ NPV (Net Present Value)
✔ 개념

👉 미래 현금흐름의 현재 가치

𝑁𝑃𝑉 = ∑((𝐶𝑎𝑠ℎ 𝐹𝑙𝑜𝑤)𝑡 / (1 + 𝑟)𝑡)

✔ 의미

“지금 기준으로 얼마 벌었는가”

✔ 예시

투자: 100억
NPV = +20억

👉 “가치 창출”

✔ 해석 기준

NPV	의미

> 0	투자 가치 있음
= 0	손익분기
< 0	손실

✔ PF에서 중요 포인트

할인율 = 자본비용(WACC)
금리 상승 → NPV 하락

✔ 위험 신호 🚨

NPV 음수 → 프로젝트 실패
금리 상승 시 급격히 악화

📊 4️⃣ 세 지표 관계

지표	역할	성격
분양률	현금 유입	운영
IRR	수익성	투자
NPV	절대 가치	재무

📌 5️⃣ 실무에서 같이 보는 이유

✔ 케이스 1

분양률 ↑ / IRR ↓

👉 많이 팔지만 수익성 낮음

✔ 케이스 2

IRR ↑ / 분양률 ↓

👉 이론상 좋지만 실제 현금 없음

✔ 케이스 3

NPV < 0

👉 무조건 재검토

👉 세 개를 같이 봐야 진짜 판단 가능

📌 6️⃣ 시스템 설계 반영


테이블 예시

pf_project
- deal_id
- total_units
- sold_units

pf_cashflow
- deal_id
- period
- cash_in
- cash_out

계산 항목
분양률 → 단순 집계
IRR → 반복 계산 (UDF 필요)
NPV → 할인 계산

📌 7️⃣ 한 줄 정리

👉

분양률 = 돈 들어오는 속도
IRR = 수익률
NPV = 실제 가치

👉 PF는 결국
“팔려야 돈이 들어오고 → 그 돈이 수익이 되는 구조”

원하면 다음 단계도 이어서 가능해요:

👉 IRR/NPV SQL or 함수 구현 (MySQL/PostgreSQL)
👉 PF Cashflow 모델 설계
👉 분양률 기반 리스크 알림 로직
