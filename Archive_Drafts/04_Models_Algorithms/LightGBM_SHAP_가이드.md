# LightGBM + SHAP 개념 정리 (초보자용)

## 1. 한 줄 핵심

-   **LightGBM = 예측하는 두뇌**
-   **SHAP = 예측 이유를 설명하는 도구**

------------------------------------------------------------------------

## 2. 문제 정의

이 딜(IPO/유증/CB)은 성공할까?

-   입력: 여러 Feature (숫자 데이터)
-   출력: 성공 확률

------------------------------------------------------------------------

## 3. Feature란?

판단 근거 데이터

예: - discount_rate (할인율) - debt_ratio (부채비율) - size_ratio
(발행규모 / 시총)

------------------------------------------------------------------------

## 4. LightGBM 개념

-   의사결정 나무(Decision Tree)를 많이 만들어서 결론 도출
-   여러 모델이 투표하는 구조

예: if 할인율 \> 10% → 성공 확률 ↑

------------------------------------------------------------------------

## 5. LightGBM 특징

-   비선형 데이터 처리 강함
-   금융 데이터에 적합
-   단점: 결과 설명 어려움

------------------------------------------------------------------------

## 6. SHAP 개념

각 Feature가 결과에 얼마나 영향을 줬는지 계산

예: - 기본값: 0.50 - 할인율: +0.20 - 부채비율: -0.10 - 시장상승: +0.12 →
최종: 0.72

------------------------------------------------------------------------

## 7. LightGBM + SHAP 관계

-   LightGBM → 예측
-   SHAP → 설명

------------------------------------------------------------------------

## 8. 전체 흐름

1.  데이터 수집
2.  Feature 생성
3.  모델 학습 (LightGBM)
4.  예측
5.  설명 (SHAP)

------------------------------------------------------------------------

## 9. 실무 핵심

-   Feature 품질이 가장 중요
-   SHAP은 금융에서 필수
-   데이터 누수 금지

------------------------------------------------------------------------

## 10. 한 줄 요약

LightGBM은 판단하고, SHAP은 그 이유를 설명한다.
