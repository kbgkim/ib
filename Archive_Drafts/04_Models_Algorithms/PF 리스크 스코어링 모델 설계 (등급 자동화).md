PD / LGD / EL을 실제로 쓰는 단계인
👉 PF 리스크 스코어링 모델 (등급 자동화)를 “설계 관점”으로 풀어드릴게요.

📌 1️⃣ 목표 정의

👉 이 모델의 목적은 단 하나입니다:

👉 Deal을 자동으로 위험등급으로 분류

입력 (지표들) → 점수화 → 등급 (AAA ~ D)

📌 2️⃣ 전체 구조

① 입력 데이터
② 지표 표준화
③ 점수 계산
④ 가중합
⑤ 등급 매핑

📊 3️⃣ 입력 변수 설계 (PF 핵심)

✔ 반드시 들어가야 하는 변수 🔥

구분	변수	의미
사업성	분양률	현금 유입
진행	공정률	사업 진행도
재무	DSCR	상환 능력
담보	LTV	회수 가능성
시장	지역지표	외부 환경
구조	트랜치	손실 흡수 위치

📌 4️⃣ 변수 점수화 (Scorecard)

👉 핵심: 모든 지표를 0~100 점으로 변환

✔ 예시: 분양률

90% 이상 → 100점
80~90% → 80점
60~80% → 60점
60% 미만 → 30점

✔ 예시: LTV

50% 이하 → 100점
50~70% → 80점
70~90% → 50점
90% 이상 → 20점

✔ 예시: DSCR

1.5 이상 → 100점
1.2~1.5 → 80점
1.0~1.2 → 50점
1.0 미만 → 20점

👉 이렇게 모든 지표를 점수화

📌 5️⃣ 가중치 설계 (핵심🔥)

👉 PF는 가중치가 매우 중요

✔ 예시 가중치

항목	가중치
분양률	30%
LTV	25%
DSCR	20%
공정률	15%
시장	10%

👉 이유:

👉 분양률 = PF의 생명

📊 6️⃣ 총점 계산

총점 = Σ (지표 점수 × 가중치)

✔ 예시

분양률: 80점 × 30% = 24
LTV: 70점 × 25% = 17.5
DSCR: 60점 × 20% = 12
공정률: 90점 × 15% = 13.5
시장: 80점 × 10% = 8

👉 총점 = 75점

📌 7️⃣ 등급 매핑

✔ 기준 테이블

점수	등급	의미
90~100	AAA	매우 안전
80~90	AA	안정
70~80	A	양호
60~70	BBB	보통
50~60	BB	위험
50 미만	B 이하	고위험

👉 예시: 75점 → A 등급

📌 8️⃣ PD로 변환 (고급🔥)

👉 등급 → PD 매핑

등급	PD
AAA	0.1%
AA	0.5%
A	1%
BBB	3%
BB	7%
B	15%

👉 자동으로 EL 계산 연결 가능

📌 9️⃣ 시스템 설계 (DB)

✔ 입력 테이블

pf_risk_input
- deal_id
- sales_rate
- ltv
- dscr
- progress_rate
- region_score

✔ 결과 테이블

pf_risk_score
- deal_id
- total_score
- rating
- pd
- lgd
- el

📌 🔟 SQL 예시 (간단)

SELECT
    deal_id,
    (sales_score * 0.3 +
     ltv_score * 0.25 +
     dscr_score * 0.2 +
     progress_score * 0.15 +
     region_score * 0.1) AS total_score
FROM pf_score_detail;

📌 11️⃣ 실무 고도화 포인트 🔥

✔ 1. 시계열 반영

분양률 추이
금리 변화

✔ 2. Trigger Alert

분양률 < 60%
OR DSCR < 1
→ 자동 경고

✔ 3. 트랜치 반영

Senior → 점수 보정 (+)
Equity → 점수 감점 (-)
