# LightGBM 내부 개념 (초보자 직관 버전)

---

# 🎯 핵심 목표

> **“틀린 만큼 보정하고, 너무 과하게 고치지 않도록 조절하는 과정”**

---

# 🧠 1️⃣ 기본 개념

LightGBM은 다음 3가지 요소로 구성됩니다.

- Gradient → 어디로 고칠지
- Hessian → 얼마나 고칠지
- Tree → 어떻게 고칠지

---

# 🔥 2️⃣ Gradient (기울기)

## ✔ 정의

```
Gradient = 예측값 - 실제값
```

## ✔ 예시

| 실제 | 예측 | Gradient | 의미 |
|------|------|----------|------|
| 1 (부도) | 0.3 | -0.7 | 더 올려야 함 |
| 0 (정상) | 0.8 | +0.8 | 더 내려야 함 |

## ✔ 핵심

> Gradient = “얼마나 틀렸고, 어느 방향으로 고쳐야 하는지”

---

# 🧊 3️⃣ Hessian (2차 미분)

## ✔ 정의

> “얼마나 신중하게 고칠지 결정하는 값”

## ✔ 직관

| 상황 | 의미 |
|------|------|
| 확실한 데이터 | 크게 수정 가능 |
| 불확실한 데이터 | 조금만 수정 |

## ✔ 핵심

> Hessian = “수정 강도 조절 장치”

---

# 🌳 4️⃣ Tree 역할

트리는 다음을 학습합니다:

```
어떤 조건일 때 더 틀렸는가?
```

## ✔ 예시

```
부채비율 > 50% AND 매출 감소 → 부도 위험 증가
```

## ✔ 핵심

> Tree = “틀린 패턴을 찾아내는 역할”

---

# 🔁 5️⃣ 전체 학습 흐름

```
1. 초기 예측 (대충 시작)
2. 틀린 정도 계산 (Gradient)
3. 수정 방향 결정
4. 수정 강도 조절 (Hessian)
5. Tree로 보정
6. 반복
```

---

# 📊 6️⃣ 실제 예시 (신용평가)

## 상황

- 실제: 부도 (1)
- 예측: 0.3

## 과정

```
1. 틀림 → Gradient 발생
2. 트리가 패턴 학습
3. 예측값 수정
```

## 변화

```
0.3 → 0.45 → 0.6 → 0.75 → ...
```

---

# 🧠 7️⃣ 한 번에 이해 (비유)

## 🎯 LightGBM = 시험 오답노트

```
1. 문제 풀기 (예측)
2. 틀린 문제 확인 (Gradient)
3. 왜 틀렸는지 분석 (Tree)
4. 수정 (Hessian으로 조절)
5. 반복
```

---

# 💡 8️⃣ 핵심 요약

- Gradient → 방향과 크기
- Hessian → 수정 강도
- Tree → 수정 방법

---

# 🚀 최종 정리

> **LightGBM = 틀린 값을 점점 줄여가는 반복 학습 시스템**

---

# 🔧 실무 연결 (신용평가)

```
LightGBM → PD (부도 확률) 계산
Gradient → 오차 기반 학습
Hessian → 안정성 확보
```
